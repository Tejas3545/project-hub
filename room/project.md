# Project Hub - FastAPI Backend Port

## 1. Project Overview
This project involves porting the existing Node.js/Express backend of **Project Hub** to Python using **FastAPI**. 
Project Hub is a platform featuring 992 curated GitHub repositories across 5 professional domains, providing developers with real-world applications to learn from also file realtime repository updates. with trending and all the language filter, project type filter, difficulty filter, and all the other filters.

The primary goal of this backend is to serve these projects to the frontend, manage user progress, handle authentication, and track analytics and gamification metrics.

## 2. Core Requirements & User Rules
The following strict rules MUST be followed during the implementation of this backend:

1. **Follow the document**: Strictly adhere to this `project.md` blueprint.
2. **Docstrings & Error Handling**: Every function, route, and class MUST have a proper Python docstring. Comprehensive error handling (using FastAPI `HTTPException`) is mandatory.
3. **Standardized Code**: Follow PEP-8 standards. Code must be typed using Python type hints (`typing` module / `pydantic`).
4. **Structured Architecture**: Organize code in a proper folder structure (see Section 3).
5. **Dependency Management (`uv`)**: Use `uv add <package>` or CLI commands for dependencies. **Do not** manually edit `pyproject.toml` or lock files.
6. **Database Schema & APIs**: Fully understand the Prisma schema (`backend/prisma/schema.prisma`) and replicate it using an asynchronous Python ORM (e.g., SQLAlchemy 2.0 or SQLModel with Asyncpg). Ensure proper fields and data validation using Pydantic.
7. **Comprehensive Testing**: After complete creation, write and execute tests for the database operations and all API endpoints. Use `pytest` and `httpx`.
8. **OpenAPI Generation**: The OpenAPI (Swagger) specification must be auto-generated by FastAPI. **Do not** write OpenAPI/Swagger files manually.
9. **Modern Features**: If unfamiliar with modern Python/FastAPI features, search the internet to implement best practices.
10. **Validation**: Validate all created APIs using the created test files once the service is up.

## 3. Project Structure
The Python backend should follow a modern, scalable FastAPI structure:

```text
backend-python/
├── src/
│   ├── main.py                 # FastAPI application instance & middlewares
│   ├── core/
│   │   ├── config.py           # Pydantic BaseSettings for env vars
│   │   ├── security.py         # JWT hashing, OAuth2 schemes
│   │   └── exceptions.py       # Custom exception handlers
│   ├── api/
│   │   ├── dependencies.py     # Auth, DB sessions, rate limiting
│   │   └── routes/             # API Router definitions
│   │       ├── auth.py
│   │       ├── users.py
│   │       ├── projects.py
│   │       ├── domains.py
│   │       ├── workspace.py
│   │       └── gamification.py
│   ├── models/                 # SQLAlchemy Database Models (replicated from Prisma)
│   ├── schemas/                # Pydantic models for Req/Res validation
│   └── services/               # Business logic, abstracted from routes
├── tests/
│   ├── conftest.py             # Pytest fixtures (DB, client)
│   ├── test_auth.py
│   ├── test_projects.py
│   └── test_users.py
├── alembic/                    # Database migrations
└── alembic.ini
```

## 4. Key Features & API Endpoints

### 4.1. Authentication & Users
- **JWT & OAuth**: Standard Email/Password + Google OAuth (via verify token).
- **Endpoints**: 
  - `POST /api/auth/register`, `POST /api/auth/login`, `POST /api/auth/google`
  - `GET /api/users/me`, `PUT /api/users/profile`

### 4.2. Domains & Projects
- **Domains**: 5 Domains (Web, AI, ML, Data Science, Cyber).
  - `GET /api/domains`
- **Projects**: Core read-only listing with robust filtering.
  - `GET /api/projects` (Paginated, filtered)
  - `GET /api/projects/{id}`
  - `GET /api/projects/search`

### 4.3. tracking user likes and bookmarks
- **Bookmarks & Likes**:
  - `POST /api/projects/{id}/bookmark`
  - `POST /api/projects/{id}/like`

### 4.4. Workspace (Notes & Deadlines)
- **Notes**: `GET/POST/PUT /api/workspace/notes`
- **Deadlines**: `GET/POST/PUT /api/workspace/deadlines`

## 5. Implementation Steps (The Plan)
1. **Initialization**:
   - Initialize the `backend-python` directory.
   - Use `uv init` and set up the virtual environment.
   - Install core dependencies: `uv add fastapi uvicorn sqlalchemy asyncpg alembic pydantic pydantic-settings pyjwt passlib bcrypt pytest httpx`.
2. **Database Setup**:
   - Translate `backend/prisma/schema.prisma` into SQLAlchemy async models.
   - Set up `alembic` for migrations.
3. **Pydantic Schemas**:
   - Define exact API inputs and outputs.
4. **Core Security & Auth**:
   - Implement JWT signing/verifying and password hashing.
   - Set up Dependency Injection for `get_current_user` and `get_db`.
5. **Route Implementation**:
   - Build out `api/routes/` and map them to `services/`.
   - Ensure rate limiting is implemented (similar to Express `express-rate-limit`).
6. **Testing & Validation**:
   - Write comprehensive unit tests in `/tests`.
   - Execute tests to ensure 100% route coverage and DB integrity.
